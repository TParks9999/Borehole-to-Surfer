# Security Policy

## Supported Versions

Security fixes are applied on a best-effort basis to the latest code on the default branch.

## Reporting a Vulnerability

Do not open a public GitHub issue for a suspected security vulnerability.

Report it privately to the repository maintainer with:

- a clear description of the issue
- affected files or workflow area
- reproduction steps or proof of concept
- impact assessment if known

Include enough detail to reproduce the issue safely. Public disclosure should wait until the issue has been reviewed and a fix or mitigation is available.
